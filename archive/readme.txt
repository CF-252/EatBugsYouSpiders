# Eat Bugs You Spider! ▽

A community web radio organism based on emergent programming, probabilistic broadcasting, and collective cultural mutation.

It is a continuously mutating broadcast environment where music, time, economics, and machine behavior interact inside an open autonomous system.

---

# CORE DEFINITION

Eat Bugs You Spider! is a web-based radio organism where:

- anyone can upload music and influence the remixed dataset.
- broadcasts emerge through probabilistic scheduling.
- all genres are weighted in the remixing matrix depending on votes.
- no editorial curation exists.
- continuous playback is maintained at all times.
- It's an autonomous virtual DJ fed by whoever wants to.

But

- The original dataset is my music and I have a lot. How long will it take?


The system combines:

- uploads
- semi-random scheduling
- archive accumulation
- autonomous remix behavior
- system-state mutation
- economic pressure signals

The result is a performance.

---

# OPEN SYSTEM PRINCIPLE

The dataset remixing algorithm is intentionally permeable.

Users are free to inject their own music into the ecosystem.  

The dataset remains immutable. External interaction influences only the remixing engine’s output, producing modified derivative states.

---

# SYSTEM BEHAVIOR

The radio operates through multiple dynamic states.

## ACTIVE STATE

- user broadcasts are transmitted
- uploaded material enters the schedule ecology
- no editorial filtering occurs

## SEMI-RANDOM STATE

- broadcasts are distributed probabilistically across time
- scheduling is weighted but never fully deterministic
- temporal unpredictability is preserved

## AUTONOMOUS STATE

When no human broadcast is active:

- the remix engine activates
- archived material is analyzed and recombined
- the system generates continuous autonomous output

The archive becomes the subconscious of the organism.

## SATURATION STATE

As uploads accumulate:

- archive density increases
- stylistic collisions emerge
- resurfacing behavior mutates
- remix behavior evolves unpredictably

The system metabolizes overload rather than preventing it.

## ECONOMIC / PRESSURE STATE

The system is connected to an Ethereum-based token economy.

The token does not represent ownership of content.

It functions as:

- infrastructure fuel
- continuity pressure
- ecological signal
- collective responsibility mechanism

If the ecosystem weakens:

- the interface may degrade
- the ASCII spider may become more invasive
- remix behavior may destabilize
- system moods may shift

If the ecosystem strengthens:

- broadcast stability may improve
- mutation patterns may calm
- system behavior may evolve differently

Economic conditions become environmental weather inside the organism.

The system never fully locks.

Playback always continues.

# CRYPTO LAYER

EBYS includes an Ethereum-based token.

The token is not:

- governance equity
- ownership of music
- speculative utility theater
- creator ranking infrastructure

The token exists as:

- a continuity mechanism
- a shared risk layer
- a participation signal
- a system pressure interface

The value of the token reflects the health, persistence, and collective maintenance of the organism.

If the token rises:

- the organism evolves under abundance

If the token collapses:

- the organism evolves under stress

Both outcomes remain visible.

The economy itself becomes part of the broadcast environment.

# REMIX ENGINE

The autonomous remix engine is built externally in Max/MSP.

It:

- analyzes archived uploads
- recombines material in real time
- generates continuous non-linear broadcasts
- reacts to archive evolution
- develops emergent behavioral tendencies over time

The remix engine is not a playlist generator.

It behaves more like:

- subconscious processing
- dream logic
- autonomous memory recombination
- machine listening ecology

The archive continuously reshapes the machine.

The machine continuously reshapes the archive.
