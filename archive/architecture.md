## 1. Project Purpose

EBYS_INFRA is a modular audio processing and AI-assisted remix system.

It combines:
- Audio separation (stems generation)
- File-based workflow automation
- AI reasoning layer (Cricket)
- Experimental remix and performance tools (Max/MSP integration)

The goal is to build a system that can transform raw audio into structured remix material with AI guidance.

---

## 2. Core Concept

Audio enters the system → gets processed → becomes structured musical material.

The pipeline is:

raw_uploads → processing → stems → analysis → remix ideas → performance tools

---

## 3. Folder Structure

### /raw_uploads
Input folder for audio files (WAV, MP3, MP4).

Purpose:
- Incoming recordings
- Tracks to be processed
- User uploads

---

### /stems
Output folder containing separated audio tracks.

Generated by:
- Demucs separation engine

Typical contents:
- drums.wav
- bass.wav
- vocals.wav
- other.wav

---

### /analysis (future / optional)
Stores metadata about audio:
- BPM
- key detection
- structural markers
- AI-generated insights

---

### /scripts
Python tools for processing:
- Demucs automation
- file watching
- AI interface scripts

---

## 4. AI Layer (Cricket)

Cricket is the AI reasoning layer of EBYS.

It:
- Reads project structure
- Understands audio pipeline
- Suggests transformations
- Explains system architecture
- Helps design remix strategies

Cricket does NOT directly process audio — it guides the system.

---

## 5. Audio Processing Engine

The system uses:

- Demucs for stem separation
- Python scripts for automation
- Optional FFmpeg for format conversion

---

## 6. Max/MSP Integration (Future Layer)

Max/MSP will be used for:
- Live audio slicing
- Performance manipulation
- Interactive remixing
- Real-time control of stems

---

## 7. Automation Status

Current state:
- Manual file placement in /raw_uploads
- Manual execution of processing scripts
- AI assistance is interactive (not autonomous)

Future state:
- Folder watching
- Auto-processing pipeline
- AI-triggered actions

---

## 8. System Philosophy

This project treats audio as data that can be:
- separated
- analyzed
- recombined
- interpreted by AI

The goal is to build a creative machine, not just a toolchain.
